**문서 버전:** v0.2

**작성자:** 개인 프로젝트
**대상 독자:** 개발자(본인), 향후 기여자

## 1. 프로젝트 개요

### 1.1 배경 및 목적

개인 투자자가 주식 시장을 효율적으로 모니터링하기 위한 macOS 전용 데스크톱 애플리케이션이다. 현재는 키움증권(영웅문) Open API를 기반으로 시작하지만, 추후 다양한 국내외 금융사 API를 플러그인 방식으로 추가할 수 있도록 확장 가능한 아키텍처를 설계한다.

백그라운드에서 실시간으로 시세를 감시하고, 사전에 설정한 조건이 충족될 때 macOS 네이티브 알림을 통해 즉시 사용자에게 알린다.

### 1.2 핵심 가치 제안

| 항목 | 설명 |
|------|------|
| **실시간 감시** | 장 중 주가·거래량을 백그라운드에서 지속 모니터링 |
| **조건부 알림** | 다양한 트리거 조건을 조합해 개인화된 알림 설정 |
| **포트폴리오 통합** | 보유 종목 기반 수익률 실시간 추적 |
| **멀티 브로커 확장** | 키움 외 증권사 추가를 위한 플러그인 구조 |

### 3.3 알림(Notification) 기능 ← 핵심 기능

모든 알림은 **macOS 네이티브 알림(UNUserNotificationCenter)** 을 통해 발송한다. 앱이 백그라운드 상태에서도 동작해야 한다.

#### 3.3.1 알림 트리거 유형

**① 특정 가격 도달 (목표가 / 손절가)**

```
조건: 현재가 >= 목표가  →  "삼성전자 목표가 85,000원 도달!"
조건: 현재가 <= 손절가  →  "삼성전자 손절가 70,000원 이탈!"
```

- 종목별 복수의 알림 가격 설정 가능 (예: 목표가 3단계)
- 알림 발생 후 자동 비활성화 or 유지 여부 선택 가능
- 원화(KRW) 및 정수 입력 지원

**② 등락률 기준 (%)**

```
조건: 등락률 >= +5%   →  "NAVER +5.2% 급등 알림"
조건: 등락률 <= -3%   →  "카카오 -3.1% 급락 알림"
```

- 기준점: 전일 종가 대비 등락률
- 임계값을 0.1% 단위로 설정 가능
- 상승/하락 독립적으로 임계값 설정

**③ 거래량 급증**

```
조건: 거래량 >= 전일 평균 거래량의 N배  →  "HMM 거래량 폭발! 평소의 5배"
```

- 기준: 최근 5일 평균 거래량 대비 배수
- 배수 임계값: 사용자 정의 (기본값 3배)
- 분봉 기준 거래량 급증 감지 옵션

**④ 뉴스 / 공시 이벤트**

```
조건: 해당 종목 공시 발생  →  "삼성전자 신규 공시: [주요사항보고서]"
```

- DART(전자공시시스템) 또는 브로커 API의 공시 피드 활용
- 공시 종류 필터링 (실적 발표, 유상증자, 자사주 매입 등)
- 뉴스 키워드 필터 설정 (예: "실적", "인수", "계약")

> **⚠️ 구현 난이도 주의:** 뉴스/공시 알림은 브로커 API 지원 여부에 따라 구현 방식이 달라짐. DART Open API 병행 연동을 검토할 것.

**⑤ 포트폴리오 수익률 변화**

```
조건: 오늘 평가손익 >= +100만원  →  "오늘 포트폴리오 +1,050,000원 달성!"
조건: 오늘 수익률 <= -2%        →  "포트폴리오 경고: -2.1% 손실"
```

- 전체 포트폴리오 기준 또는 종목별 기준 선택 가능
- 오늘 손익 / 전체 손익 두 가지 기준 제공
- 금액(원) 또는 비율(%) 기준 선택

#### 3.3.2 알림 공통 설정

- **알림 활성화 시간대:** 장 시간(09:00~15:30)만 발송 / 24시간 발송 선택
- **쿨다운:** 동일 조건 재발송 최소 간격 설정 (기본 5분)
- **알림 소리:** macOS 시스템 사운드 선택 또는 무음
- **알림 클릭 시 동작:** 앱 포커스 / 해당 종목 상세 페이지 오픈

### 3.4 포트폴리오 관리

#### 3.4.1 포트폴리오 등록
- 보유 종목, 평균 매입가, 수량 수동 입력
- 키움 API 연동 시 잔고 자동 불러오기 (선택)

#### 3.4.2 수익률 대시보드
- 종목별 평가손익, 수익률(%) 실시간 표시
- 전체 포트폴리오 총 평가금액, 총 손익 요약
- 매입가 대비 현재 수익률 시각화 (바 게이지)

#### 3.4.3 포트폴리오 자산 변화 추이 (기간별 비교)

포트폴리오 총 평가금액의 시간 흐름에 따른 변화를 꺾은선 그래프로 시각화한다. 특정 시점과 현재를 비교해 자산이 얼마나 변했는지 직관적으로 확인하는 것이 목적이다.

**비교 단위**

| 단위 | 비교 기준 예시 |
|------|---------------|
| 일(Day) | 어제 vs 오늘 / 특정 날짜 vs 오늘 |
| 주(Week) | n월 m째 주 시작 vs 현재 |
| 월(Month) | n월 1일 vs 현재 |
| 연(Year) | 올해 1월 1일 vs 현재 |

**데이터 수집 방식 (v1 — 간단·유연 방식)**

- 앱 실행 중 일정 주기(기본값: 1분)마다 포트폴리오 총 평가금액을 **스냅샷**으로 SQLite에 저장
- 앱이 종료된 동안은 스냅샷이 쌓이지 않으며, 해당 구간은 그래프에서 공백(gap)으로 표시
- 저장 데이터 구조:

```
portfolio_snapshots
  - id          INTEGER PRIMARY KEY
  - timestamp   DATETIME          -- 스냅샷 기록 시각
  - total_value INTEGER           -- 총 평가금액 (원)
  - total_gain  INTEGER           -- 총 평가손익 (원)
  - gain_pct    REAL              -- 총 수익률 (%)
```

**그래프 표시 방식**

- X축: 시간 (선택한 기간에 따라 자동 스케일 조정)
- Y축: 총 평가금액(원) 또는 수익률(%)  — 토글로 전환
- 비교 기준 시점(예: 이번 주 월요일)을 수평 기준선(baseline)으로 표시
- 기준선 대비 현재 금액 차이 및 변화율을 그래프 상단에 요약 표시

```
예시: 이번 주 비교
─────────────────────────────────────────────
 이번 주 (5/12 월 ~ 오늘)       +850,000원 (+1.7%)
─────────────────────────────────────────────
  ₩52M ┤                              ╭─●
  ₩51M ┤               ╭─────────────╯
  ₩50M ┤──────────╮────╯   ← 기준선 (월요일 시가)
  ₩49M ┤          ╰─╮
       └─────────────────────────────────────
        월         화        수       목(오늘)
```

**기간 선택 UI**

- 상단 세그먼트 컨트롤: `일` · `주` · `월` · `연`
- 각 단위 선택 시 하위 드롭다운으로 세부 기간 지정
  - 일: 날짜 피커 (어제, 오늘, 특정 날짜)
  - 주: "n월 m째 주" 선택
  - 월: 월 선택
  - 연: 연도 선택

**향후 개선 가능 항목 (v2+)**

- 스냅샷 주기 사용자 설정 (1분 / 5분 / 10분)
- 종목별 개별 자산 추이 비교
- 데이터 보존 기간 설정 및 오래된 스냅샷 자동 정리
- 일/주 단위는 장 마감 시 자동 집계 포인트 생성

### 5.2 브로커 어댑터 구조 (확장 설계)

멀티 브로커 지원을 위해 **Protocol(인터페이스) 기반 어댑터 패턴**을 사용한다.

```swift
protocol BrokerAdapter {
    var brokerName: String { get }
    func connect(credentials: BrokerCredentials) async throws
    func fetchQuote(symbol: String) async throws -> StockQuote
    func fetchPortfolio() async throws -> [PortfolioItem]
    func subscribeRealtime(symbols: [String], handler: @escaping (StockQuote) -> Void)
    func fetchNews(symbol: String) async throws -> [NewsItem]
}
```

새로운 증권사 추가 시 `BrokerAdapter` 프로토콜을 구현하는 클래스만 추가하면 된다.

|--------|-----------|

| 알림 조건 설정 | UserDefaults 또는 SQLite |
| 알림 이력 | SQLite |
| **포트폴리오 스냅샷 이력** | **SQLite (portfolio_snapshots 테이블)** |
| 앱 설정 | UserDefaults |

## 6. 기술 스택 (권고)

| 영역 | 기술 | 비고 |
|------|------|------|
| 언어 | Swift 5.9+ | macOS 네이티브 |

| 차트 | Swift Charts (iOS 16+ / macOS 13+) | 자산 변화 꺾은선 그래프 |

> **참고:** 키움 영웅문 Open API는 Windows 전용 COM 방식과 REST API 방식이 공존한다. macOS에서는 **KIS Developers (한국투자증권) REST API** 또는 **키움 REST API** 활용을 검토할 것. 영웅문 COM 방식은 macOS에서 직접 사용 불가능.

### 7.2 설정 화면 탭 구성

1. **계좌 연결** — 브로커 선택 / API 키 입력 / 연결 테스트
2. **관심 종목** — 종목 추가/삭제, 그룹 관리
3. **알림 설정** — 종목별 알림 조건 목록, 추가/편집
4. **포트폴리오** — 보유 종목 및 매입가 입력
5. **자산 추이** — 기간별 자산 변화 꺾은선 그래프
6. **앱 설정** — 갱신 주기, 장 시간 설정, 알림 소리 등

## 9. 제약사항 및 리스크

| 항목 | 내용 | 대응 방안 |
|------|------|-----------|
| 키움 API macOS 미지원 | 영웅문 HTS용 COM API는 Windows 전용 | 키움 REST/WebSocket API 활용, 불가 시 Wine/VM 방식 검토 |
| 실시간 시세 제한 | API 호출 빈도 제한 존재 | WebSocket 구독 방식 우선 활용, 폴링 주기 조정 |
| 공시 알림 지연 | 외부 API 지연으로 실시간성 한계 | 폴링 주기 최소화, 사용자에게 지연 가능성 명시 |
| Apple 공증 (Notarization) | 배포 시 필요 | 개인 사용 목적이면 로컬 빌드로 우선 대응 |
