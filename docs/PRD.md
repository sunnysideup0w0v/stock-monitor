## 1. 프로젝트 개요

### 1.1 배경 및 목적

개인 투자자가 주식 시장을 효율적으로 모니터링하기 위한 macOS 전용 데스크톱 애플리케이션이다. **한국투자증권(KIS) Developers REST/WebSocket API**를 주 브로커로 사용하며, 추후 키움증권·미래에셋증권 등 다양한 증권사 API를 플러그인 방식으로 추가할 수 있도록 확장 가능한 아키텍처를 설계한다.

백그라운드에서 실시간으로 시세를 감시하고, 사전에 설정한 조건이 충족될 때 macOS 네이티브 알림을 통해 즉시 사용자에게 알린다.

### 1.2 핵심 가치 제안

| 항목 | 설명 |
|------|------|
| **실시간 감시** | 장 중 주가·거래량을 백그라운드에서 지속 모니터링 |
| **조건부 알림** | 다양한 트리거 조건을 조합해 개인화된 알림 설정 |
| **포트폴리오 통합** | 보유 종목 기반 수익률 실시간 추적 |
| **멀티 브로커 확장** | 증권사 추가를 위한 플러그인 구조 |

### 3.2 종목 관리

#### 3.2.1 관심 종목 등록
- 종목 코드 및 이름 직접 입력으로 관심 목록에 추가

#### 3.2.2 시세 표시

- 거래량 표시

### 3.3 알림(Notification) 기능 ← 핵심 기능

모든 알림은 **macOS 네이티브 알림(UNUserNotificationCenter)** 을 통해 발송한다. 앱이 백그라운드 상태에서도 동작해야 한다.

#### 3.3.1 알림 트리거 유형

**① 특정 가격 도달 (목표가 / 손절가)**

```
조건: 현재가 >= 목표가  →  "삼성전자 목표가 85,000원 도달!"
조건: 현재가 <= 손절가  →  "삼성전자 손절가 70,000원 이탈!"
```

- 종목별 복수의 알림 가격 설정 가능 (예: 목표가 3단계)
- 알림 발생 후 자동 비활성화 or 유지 여부 선택 가능
- 원화(KRW) 및 정수 입력 지원

**② 등락률 기준 (%)**

```
조건: 등락률 >= +5%   →  "NAVER +5.2% 급등 알림"
조건: 등락률 <= -3%   →  "카카오 -3.1% 급락 알림"
```

- 기준점: 전일 종가 대비 등락률
- 임계값을 0.1% 단위로 설정 가능
- 상승/하락 독립적으로 임계값 설정

**③ 거래량 급증**

```
조건: 거래량 >= 전일 평균 거래량의 N배  →  "HMM 거래량 폭발! 평소의 5배"
```

- 기준: 최근 5일 평균 거래량 대비 배수
- 배수 임계값: 사용자 정의 (기본값 3배)

**④ 공시 이벤트 (DART)**

```
조건: 해당 종목 공시 발생  →  "삼성전자 신규 공시: [주요사항보고서]"
```

- DART Open API(`/api/list.json`) 5분 간격 폴링으로 공시 감지
- 공시 종류 필터링 (정기공시·주요사항·발행·지분·기타·거래소)
- 알림 클릭 시 DART 공시 본문 페이지 브라우저에서 오픈

**⑤ 포트폴리오 수익률 변화**

```
조건: 평가손익 >= +100만원  →  "포트폴리오 +1,050,000원 달성!"
조건: 수익률 <= -2%        →  "포트폴리오 경고: -2.1% 손실"
```

- 전체 포트폴리오 기준
- 금액(원) 또는 비율(%) 기준 선택 — 4가지 트리거 타입 지원

#### 3.3.2 알림 공통 설정

- **알림 활성화 시간대:** 장 시간(09:00~15:30)만 발송 / 24시간 발송 선택
- **쿨다운:** 동일 조건 재발송 최소 간격 설정 (기본 **60분**)
- **알림 소리:** macOS 시스템 사운드 선택 또는 무음 (15종)
- **알림 클릭 시 동작:** 팝오버 오픈 / DART 공시 URL 오픈 (공시 알림 시)
- **단절 알림:** 연속 2회 폴링 실패 시 알림 발송, 복구 시 재연결 알림

### 3.4 포트폴리오 관리

#### 3.4.1 포트폴리오 등록
- 보유 종목, 평균 매입가, 수량 수동 입력
- **KIS API 연동 시 잔고 자동 불러오기** (가져오기 전 미리보기 시트, 신규 추가/전체 교체 선택)
- **계정 종속**: 포트폴리오 항목은 로그인한 계정(KIS appKey)에 종속됨. 로그아웃 시 목록이 노출되지 않음
- 팝오버 노출 여부 종목별 개별 설정 가능 (눈 아이콘 토글)

#### 3.4.2 수익률 대시보드
- 종목별 평가손익, 수익률(%) 실시간 표시
- 전체 포트폴리오 총 평가금액, 총 손익 요약

#### 3.4.3 포트폴리오 자산 변화 추이 (기간별 비교)

포트폴리오 총 평가금액의 시간 흐름에 따른 변화를 꺾은선 그래프로 시각화한다. 특정 시점과 현재를 비교해 자산이 얼마나 변했는지 직관적으로 확인하는 것이 목적이다.

**비교 단위**

| 단위 | 비교 기준 예시 |
|------|---------------|
| 일(Day) | 어제 vs 오늘 / 특정 날짜 vs 오늘 |
| 주(Week) | n월 m째 주 시작 vs 현재 |
| 월(Month) | n월 1일 vs 현재 |
| 연(Year) | 올해 1월 1일 vs 현재 |

**데이터 수집 방식**

- 앱 실행 중 1분마다 포트폴리오 총 평가금액을 **스냅샷**으로 SQLite에 저장
- 장 시간(09:00~15:30)만 수집 토글 (기본 ON), 커스텀 시간대 추가 가능
- 앱이 종료된 동안은 스냅샷이 쌓이지 않으며, 해당 구간은 그래프에서 공백(gap)으로 표시
- 데이터 보존 기간 설정 가능 (30/90/180/365일/무제한) 및 수동 정리

**그래프 표시 방식**

- X축: 시간 (선택한 기간에 따라 자동 스케일 조정)
- Y축: 총 평가금액(원) 또는 수익률(%) — 토글로 전환
- 비교 기준 시점을 수평 기준선(baseline)으로 표시
- 기준선 대비 현재 금액 차이 및 변화율을 그래프 상단에 요약 표시
- 데이터 공백 구간은 선 분리로 표현

**향후 개선 가능 항목 (v2+)**

- 스냅샷 주기 사용자 설정 (1분 / 5분 / 10분)
- 종목별 개별 자산 추이 비교

## 4. 비기능 요구사항

### 4.1 성능

| 항목 | 목표 |
|------|------|
| 시세 갱신 주기 | 실시간 (WebSocket) 또는 최소 3초 폴링 |
| 알림 발송 지연 | 조건 충족 후 3초 이내 |
| 앱 백그라운드 CPU | 평균 1% 이하 유지 목표 |
| 앱 메모리 사용 | 상시 200MB 이하 목표 |

### 4.2 안정성

- 앱 실행 시 macOS 로그인 항목에 자동 등록 (시작프로그램, SMAppService)
- 네트워크 단절 시 자동 재연결 (Exponential Backoff: 1s → 60s) 및 사용자 알림
- API 응답 오류 시 재시도 로직 (403/503 → 1회 즉시 재시도)
- 크래시 발생 시 로그 저장 (`~/Library/Logs/StockWatch/`)

### 4.3 보안

- API 키 및 계좌 정보는 macOS Keychain에 암호화 저장
- 네트워크 통신 TLS 1.2 이상 강제
- 로컬 데이터(DB) 암호화 고려

### 4.4 사용성 (UX)

- 메뉴바(Status Bar) 상주 앱 형태 — 별도 Dock 아이콘 없음
- 다크 모드 / 라이트 모드 자동 대응
- 최초 실행 시 온보딩 가이드 (5단계: 환영 → 계좌 연결 → 관심종목 → 알림 설정 → 완료)
- 로그아웃 상태에서 관심종목·포트폴리오 탭에 계좌 연결 안내 메시지 표시

### 5.2 브로커 어댑터 구조 (확장 설계)

멀티 브로커 지원을 위해 **Protocol(인터페이스) 기반 어댑터 패턴**을 사용한다. 어댑터는 `actor`로 구현해 토큰 캐시 등 내부 상태의 thread safety를 보장한다.

```swift
protocol BrokerAdapter: Sendable {
    var brokerName: String { get }
    func connect(credentials: BrokerCredentials) async throws
    func disconnect() async
    func fetchQuote(symbol: String) async throws -> StockQuote
    func fetchPortfolio() async throws -> [PortfolioItem]
    func fetchNews(symbol: String) async throws -> [NewsItem]
    func fetchDailyVolumes(symbol: String, days: Int) async throws -> [Int]
}
```

새로운 증권사 추가 시 `BrokerAdapter` 프로토콜을 구현하는 `actor`만 추가하면 된다.  
`BrokerRegistry`가 브로커 이름 기반으로 어댑터를 등록·조회·해제한다.

| API 키 / 계좌 정보 | macOS Keychain |

| 앱 설정 / 필터 설정 | UserDefaults |

2. **관심종목** — 종목 추가/삭제, 그룹·별칭 설정 (로그아웃 시 계좌 연결 안내)
3. **포트폴리오** — 보유 종목·매입가·수량 입력, KIS 잔고 자동 가져오기 (로그아웃 시 계좌 연결 안내)
4. **알림설정** — 종목별·포트폴리오 알림 조건 목록, 추가/편집
5. **알림 이력** — 날짜 범위·종목·유형 필터, CSV 내보내기, DART 공시 링크
6. **자산 차트** — 일/주/월/연 기간별 자산 변화 꺾은선 그래프

## 9. 제약사항 및 리스크

| 항목 | 내용 | 대응 방안 |
|------|------|-----------|
| KIS API 호출 제한 | 초당 호출 수 제한 존재 | WebSocket 구독 방식 우선, 폴링 주기 3초 유지 |
| 실시간 시세 제한 | WebSocket 연결 유지 필요 | 자동 재연결 (Exponential Backoff 구현 완료) |
| 공시 알림 지연 | DART API 5분 폴링 → 실시간 아님 | 사용자에게 지연 가능성 명시 |
| Apple 공증 (Notarization) | 배포 시 필요 | 개인 사용 목적이면 로컬 빌드로 대응 |
| 멀티 브로커 계정 종속 | 복수 증권사 연결 시 accountId 스키마 확장 필요 | 현재는 KIS 단일 계정 기준. 멀티 브로커 지원 시 설계 재검토 필요 |

## 10. 용어 정의

| 용어 | 설명 |
|------|------|
| 트리거 | 알림을 발생시키는 조건 |
| 관심 종목 | 모니터링할 종목 목록 (계정 종속) |
| 포트폴리오 | 현재 보유 중인 종목과 수량/매입가 정보 (계정 종속) |
| 어댑터 | 특정 증권사 API를 통합 인터페이스로 연결하는 모듈 (actor) |
| 쿨다운 | 동일 조건의 알림 재발송을 막는 최소 대기 시간 |
| 평가손익 | (현재가 - 평균 매입가) × 수량 |
| 스냅샷 | 특정 시각의 포트폴리오 총 평가금액을 기록한 데이터 포인트 |
| 기준선 | 기간 비교 그래프에서 비교 시작 시점의 자산 금액을 나타내는 수평선 |
| AccountId | 계정 식별자. `"KIS-" + appKey.prefix(8)` 형식 |
