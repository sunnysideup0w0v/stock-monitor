### 2. 에러 로깅 강화

**`CrashLogger` 시그널 핸들러 추가**

기존에는 `NSSetUncaughtExceptionHandler` (ObjC 예외만 포착)만 있었음. Swift 크래시(`SIGABRT`, `SIGILL`, `SIGSEGV`, `SIGFPE`, `SIGBUS`, `SIGTRAP`)는 signal 기반이라 별도 처리 필요.

```swift
signal(SIGABRT) { caught in
    CrashLogger.writeSignal("SIGABRT")
    signal(caught, SIG_DFL); raise(caught)  // 시스템 크래시 리포트도 생성
}
```

로그 저장 위치: `~/Library/Logs/StockWatch/crash-YYYY-MM-DD.log`

**`AppLogger` 신규 파일**

`os.Logger` 기반 구조화 로깅. Console.app + 파일 이중 기록.

```swift
AppLogger.log("ScreenerView appeared", category: "Screener")
AppLogger.log("runScreener: error=\(error)", level: .error, category: "Screener")
```

- Console.app: `subsystem:com.personal.StockWatch` 필터로 조회
- 파일: `~/Library/Logs/StockWatch/app-YYYY-MM-DD.log`

ScreenerView 로깅 포인트: `onAppear`, `onDisappear`, `autoRunScreener` 시작/완료/오류, `runScreener` 시작/완료/오류

### 4. ETF / 주식 종목 유형 필터

**DB Migration v10**: `stock_universe.isEtf` 컬럼 추가 (`BOOLEAN NOT NULL DEFAULT 0`)

**`StockUniverseItem`**: `var isEtf: Bool` 필드 추가

**`KRXManager.detectEtf(_:)`**: 이름 기반 ETF 감지

```swift
static func detectEtf(_ name: String) -> Bool {
    let prefixes = ["KODEX", "TIGER", "ARIRANG", "KINDEX", "HANARO", "KOSEF",
                    "ACE ", "SOL ", "PLUS ", "TIMEFOLIO", "히어로즈", "KB스타",
                    "BNK", "TREX", "SMART", "KIRIUM", "KTOP"]
    if prefixes.contains(where: { name.hasPrefix($0) }) { return true }
    return name.hasSuffix("ETF") || name.contains(" ETF ")
}
```

KRX OpenAPI 및 네이버 폴백 양쪽에서 `StockUniverseItem` 생성 시 적용.

**`ScreenerCondition.ConditionType.instrumentType`**: 새 케이스 추가

**`ScreenerEngine`**: `isEtf` 컬럼 필터 처리

## 설계 결정

### ETF 감지를 DB 컬럼으로 저장한 이유
SQL 쿼리 시 `WHERE isEtf = 1` 인덱스 활용 가능. 매 쿼리마다 이름 패턴 매칭을 런타임에 실행하면 4000+ 건 전체 스캔이 필요해 비효율적.

### MarkdownUI 스트리밍 방식: 완료 후 렌더링
스트리밍 중 Markdown 뷰를 실시간 업데이트하면 부분 파싱으로 렌더링이 깨짐. 로딩 인디케이터 → 완료 후 전체 렌더링이 시각적으로 더 깔끔함.

### 체크박스 다중 선택에서 stringValue 콤마 구분 사용
기존 `ScreenerCondition` 모델(`stringValue: String?`) 변경 없이 다중 선택 지원. 저장/복원(UserDefaults JSON) 구조 유지. GRDB의 `Collection.contains(Column)` 으로 자연스럽게 `IN` 쿼리 생성.
