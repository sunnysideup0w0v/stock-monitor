## 작업 내역

### Phase 3.1/3.2 — 에러 핸들링 강화 및 자동 재연결

#### `QuoteManager` — 단절 감지 및 복구 알림

```swift
private var disconnectNotified = false
static var disconnectAlertEnabled: Bool  // UserDefaults "QuoteManager.disconnectAlert", 기본 true
```

- **단절 감지**: `consecutiveFailures >= 2`이고 `disconnectNotified == false`이면 macOS 알림 1회 발송 + `disconnectNotified = true`
- **복구 감지**: 다음 폴링 성공 시 `disconnectNotified == true`이면 "재연결됨" 알림 발송 후 플래그 리셋
- **`reconnect()` 메서드 추가**: `currentSymbols`가 비어있지 않으면 `startPolling(symbols: currentSymbols)` 재호출

```swift
func reconnect() {
    guard !currentSymbols.isEmpty else { return }
    startPolling(symbols: currentSymbols)
}
```

팝오버 헤더에 "재연결" 버튼 추가 (`connectionState == .error`일 때만 표시).  
알림설정 탭에 "단절 시 알림" 토글(`disconnectAlertEnabled`) 추가.

#### `KISAdapter` — HTTP 403/503 재시도

```swift
func fetchQuote(symbol:, retryOnUnauthorized: Bool = true, retryCount: Int = 1) async throws -> StockQuote
```

403·503 응답 수신 시 `retryCount > 0`이면 1초 대기 후 `retryCount - 1`로 재귀 호출. 기존 401 재시도 로직과 별개 경로.

#### `CrashLogger` — 미처리 예외 로그 저장

```swift
enum CrashLogger {
    static func install() {
        NSSetUncaughtExceptionHandler { CrashLogger.write(exception: $0) }
    }
}
```

로그 위치: `~/Library/Logs/StockWatch/crash-YYYY-MM-DD.log`  
같은 날 여러 번 크래시 시 `---` 구분자로 추가 기록(append).  
`AppDelegate.applicationDidFinishLaunching`에서 `CrashLogger.install()` 호출.

### Phase 3.3 — macOS 로그인 시 자동 시작

`SMAppService` (macOS 13+ 권장 API)를 사용해 로그인 항목을 등록/해제한다.

```swift
import ServiceManagement

// AccountSettingsView
@State private var launchAtLogin = (SMAppService.mainApp.status == .enabled)

Toggle("로그인 시 자동 시작", isOn: $launchAtLogin)
    .onChange(of: launchAtLogin) { _, enabled in
        if enabled { try? SMAppService.mainApp.register() }
        else        { try? SMAppService.mainApp.unregister() }
    }
```

설정 → 계좌 연결 탭 하단 "유틸리티" 섹션에 토글 배치.  
유효 코드 서명이 없으면 `register()`가 실패하므로 실제 자동 시작 동작은 수동 확인이 필요하다.

### Phase 3.4 — 설정 백업/복원

**`BackupManager.swift`** (신규, `@MainActor`)

```swift
struct Backup: Codable {
    var version: Int = 1
    var exportedAt: Date
    var watchlist: [WatchlistItem]
    var portfolio: [PortfolioItem]
    var alertConditions: [AlertCondition]
}
```

- **내보내기** (`export()`): `NSSavePanel` → JSON 인코딩(prettyPrinted + iso8601) → 파일 저장
  - 기본 저장 경로: `~/Downloads`
  - 기본 파일명: `StockWatch-backup-YYYY-MM-DD.json`
- **가져오기** (`importBackup()`): `NSOpenPanel` → JSON 디코딩 → `restore(from:)` 호출
- **복원 전략**: 관심종목·포트폴리오는 심볼 중복 시 스킵(기존 항목 유지), 알림 조건은 전부 추가

계좌 연결 탭 하단에 "설정 내보내기" · "설정 가져오기" 버튼 추가.

## 기술적 결정 사항

| 결정 | 이유 |
|------|------|
| 단절 알림을 `consecutiveFailures >= 2` 기준으로 1회만 발송 | 일시적 네트워크 지연(1회 실패)에는 알림 없이 통과; 반복 실패 시에만 알림으로 노이즈 최소화 |
| `disconnectNotified` 플래그로 중복 알림 차단 | 폴링 3초 간격으로 단절 시 수십 개 알림이 쏟아지는 문제 방지 |
| 크래시 로그를 `~/Library/Logs/StockWatch/`에 저장 | macOS 표준 로그 디렉터리; Console.app에서 조회 가능 |
| `BackupManager.restore()`: 중복 심볼 스킵 전략 | "가져오기"를 병합(merge) 개념으로 설계 — 기존 수정 사항을 덮어쓰지 않음 |
| 버전 정보를 `Bundle.main.infoDictionary`에서 읽기 | `project.yml`의 `MARKETING_VERSION`이 빌드 시 `CFBundleShortVersionString`으로 자동 주입됨 |
| 테스트용 `private` → `internal` 변환 최소화 | `CrashLogger.write()`, `BackupManager.restore()` 2개만 노출; 나머지는 파라미터 오버로드 패턴 유지 |
