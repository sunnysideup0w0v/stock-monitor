### Phase 2.1 — WebSocket 실시간 시세

**목표:** KIS WebSocket API를 통해 폴링(3초) 대신 실시간 체결가 수신

이전 세션에서 작업을 시작했으나 컨텍스트가 종료되어 이번 세션에서 마무리했다.

#### 구현 내용

**`RealtimeQuoteManager.swift` (신규, actor)**

KIS WebSocket 연결의 전체 생명주기를 담당한다.

- **승인키 발급**: `POST /oauth2/Approval` — REST 토큰과 별개로 WebSocket 전용 승인키가 필요하다. 필드명이 `secretkey` (REST는 `appsecret`)로 다른 점에 주의.
- **WebSocket 연결**: 실전 `ws://ops.koreainvestment.com:21000` / 모의 `:31000`
- **구독 메시지**: `tr_id: "H0STCNT0"`, `tr_type: "1"(구독) / "2"(해제)`
- **메시지 파싱**: `{encrypt}|{tr_id}|{count}|{data}` 형식. `^` 구분자로 필드 분리.
  - `fields[0]` 종목코드, `[2]` 현재가, `[3]` 부호, `[4]` 변동폭, `[5]` 등락률, `[13]` 거래량
  - 암호화 메시지(`parts[0] == "1"`)는 현재 무시
- **지수 백오프 재연결**: 1s → 2s → 4s → … → 60s 상한
- **20초 핑**: `webSocketTask?.sendPing { _ in }` 으로 연결 유지

**`QuoteManager.swift` 변경**

```swift
func startRealtime(credentials: BrokerCredentials, isMock: Bool)
func stopRealtime()
func updateFromRealtime(quote: StockQuote)   // WebSocket 수신 시 MainActor에서 호출
```

WebSocket과 폴링이 공존하는 구조다. WebSocket이 연결되면 실시간 업데이트가 오고, 폴링도 별도로 유지되어 WebSocket 미연결 상태의 폴백 역할을 한다.

**`Info.plist` 변경**

KIS WebSocket이 `ws://`(비TLS)를 사용하므로 ATS 예외 도메인을 추가했다.

```xml
<key>NSExceptionDomains</key>
<dict>
    <key>ops.koreainvestment.com</key>
    <dict>
        <key>NSExceptionAllowsInsecureHTTPLoads</key>
        <true/>
    </dict>
</dict>
```

### 설정 UI 리팩토링 — 공통 컴포넌트 추출

**배경**: 계좌 연결 탭에만 적용되어 있던 좌우 패딩이 다른 탭(관심종목·포트폴리오·알림설정)에는 없어 일관성이 깨져 있었다.

#### SwiftUI 컴포넌트화 — React와의 비교

React의 `children` prop과 동일한 개념이 SwiftUI에서는 `@ViewBuilder`다.

```swift
// SwiftUI
struct SettingsTabContainer<Content: View>: View {
    let title: String
    let content: Content

    init(title: String, @ViewBuilder content: () -> Content) {
        self.title = title
        self.content = content()
    }

    var body: some View {
        VStack(alignment: .leading, spacing: 12) {
            Text(title).font(.title2).bold()
            content
        }
        .padding([.horizontal, .bottom], 8)
    }
}
```

`<Content: View>` 제네릭이 TypeScript의 `ReactNode` 타입 제약에 해당하고, `@ViewBuilder`가 클로저 안의 여러 뷰를 하나의 `Content`로 합쳐준다.

#### 추출한 컴포넌트

| 컴포넌트 | 역할 |
|----------|------|
| `SettingsTabContainer` | 타이틀 + 좌우하단 8px 패딩. 모든 설정 탭에 적용 |
| `SettingsFormSection` | Divider + `.headline` 섹션 제목. 종목 추가 폼 3곳에서 공유 |

## 작업 내역

### Phase 2.10 — KIS 잔고조회 연동 (보유 종목 자동 가져오기)

#### 구현 배경

기존에는 포트폴리오에 종목을 수동으로 입력해야 했다. KIS 계좌가 연결된 상태라면 잔고조회 API로 실제 보유 종목을 한 번에 가져올 수 있다.

#### KISAdapter — `fetchPortfolio()` 실구현

```
GET /uapi/domestic-stock/v1/trading/inquire-balance
tr_id: 실전 TTTC8434R / 모의 VTTC8434R
```

계좌번호 파싱: 사용자가 입력한 `"50123456-01"` → `-` 제거 후 앞 8자리를 `CANO`, 뒤 2자리를 `ACNT_PRDT_CD`로 분리. 10자리 미만이면 `ACNT_PRDT_CD = "01"` 기본값 적용.

응답 `output1` 배열에서 `hldg_qty > 0`인 항목만 `PortfolioItem`으로 변환. `pchs_avg_pric`은 소수점 포함 문자열(`"85200.00"`)이므로 `Double` → `Int`로 변환.

#### QuoteManager — `fetchBalance()` 위임 메서드

```swift
func fetchBalance() async throws -> [PortfolioItem] {
    guard let adapter else { throw BrokerError.notConnected }
    return try await adapter.fetchPortfolio()
}
```

뷰에서 직접 어댑터를 참조하지 않고 QuoteManager를 통해 호출하도록 단일 진입점 유지.

#### PortfolioSettingsView UI

- **"계좌에서 가져오기" 버튼**: KIS 연결 여부(`Keychain kis.appKey` 존재 확인)에 따라 활성/비활성 자동 전환. 불러오는 동안 스피너 표시.
- **인라인 에러 메시지**: 실패 시 버튼 옆에 오류 문구 노출.

#### PortfolioImportSheetView — 미리보기 시트

가져온 종목을 확인하고 동기화 방식을 선택하는 모달.

```
보유 종목 가져오기
계좌에서 3개 종목을 불러왔습니다.
삼성전자 · SK하이닉스 · NAVER
────────────────────────────────
삼성전자                 평균 85,200원
005930              10주 · 총 852,000원
────────────────────────────────
동기화 방식
○ 신규 추가만 — 이미 있는 종목 유지
○ 전체 교체 — 기존 항목 삭제 후 덮어쓰기

                     [취소]  [가져오기]
```

**UI 개선 과정**
1. 서브타이틀에 종목명 나열 추가 (`·` 구분자)
2. bordered `List` → Divider 사이 `ScrollView` + 수동 행으로 교체 (bordered List 하단 경계선과 명시적 Divider가 겹쳐 이중선처럼 보이는 문제 해결)
3. Divider와 목록 사이 불필요한 여백 제거: Divider + ScrollView + Divider를 `VStack(spacing: 0)`으로 묶음

### 보유 종목 팝오버 실시간 표시

#### 설계 결정: DB 컬럼 방식

종목별 팝오버 표시 여부를 저장하는 방법으로 UserDefaults 집합 대신 `portfolio` 테이블에 컬럼을 추가했다. DB와 모델이 항상 동기화되어 별도 상태 관리가 불필요하다.

**DB Migration v7**: `portfolio` 테이블에 `showInPopover BOOLEAN NOT NULL DEFAULT 0` 컬럼 추가.

#### PortfolioSettingsView — 눈 아이콘 토글

각 포트폴리오 행 왼쪽에 눈 아이콘(`eye.fill` / `eye.slash`) 버튼 추가. 클릭 시 `showInPopover` 값을 DB에 즉시 저장.

#### MenuBarPopoverView — portfolioSection

포트폴리오 요약 행 아래에 `showInPopover = true`인 종목의 현재가를 인덴트된 서브행으로 표시.

```
포트폴리오              +12,345원
  삼성전자   87,500원   +2.7%
  SK하이닉스 165,500원  -0.3%
```

`PortfolioHoldingRowView`: 종목명(secondary) + 현재가(monospaced) + 수익률%(green/red).

#### AppDelegate — 시작 시 폴링 누락 버그 수정

`startPollingFromDB()`가 watchlist 심볼만 폴링하고 있어, 팝오버를 한 번이라도 열기 전까지는 보유 종목 현재가를 가져오지 못하는 문제가 있었다. `showInPopover = true`인 포트폴리오 심볼을 watchlist 심볼과 합쳐서 앱 시작 시 즉시 폴링을 시작하도록 수정했다.

### docs/wiki/views.md — 뷰 용어 정의 문서 신규 작성

앱 내 UI 구성 요소의 공식 명칭을 통일하기 위해 `docs/wiki/` 폴더를 생성하고 `views.md`를 작성했다.

**주요 결정: 메뉴바 아이콘 클릭 시 나타나는 창의 공식 명칭 = "팝오버"**

근거: AppKit `NSPopover`를 그대로 사용하며, 코드 전체(`popover`, `MenuBarPopoverView`, `popoverWillShow`)와 명칭이 일치한다. 코드-문서 간 용어 불일치를 방지하기 위해 기술 용어를 그대로 채택.

문서 구성:
- 팝오버 / 설정 창 / 보유 종목 가져오기 시트 각각의 정의·구현 파일·역할
- 화면 표현 ↔ 코드 명칭 ↔ 공식 명칭 대조표
