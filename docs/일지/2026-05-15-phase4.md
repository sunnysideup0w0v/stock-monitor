## 작업 내역

### Phase 4.1 — BrokerAdapter 프로토콜 정비 및 BrokerRegistry

기존에 `QuoteManager`가 단일 `any BrokerAdapter`만 보유하던 구조에서 복수 어댑터를 지원할 수 있도록 기반을 마련했다.

**`BrokerRegistry`** (새 싱글턴): 현재 연결된 어댑터 목록을 관리. `register(_ adapter:)` / `unregister(brokerName:)` 메서드 제공. 설정 창 UI가 "어느 브로커가 연결됐는가"를 판단할 때 사용.

**`KiwoomAdapter`** 스텁 추가: `actor`로 구현. `brokerName = "키움증권"`. 실구현은 Phase 4.4에서 진행.

**`BrokerAdapter` 프로토콜 메서드**:
```swift
func connect(credentials: BrokerCredentials) async throws
func disconnect() async
func fetchQuote(symbol: String) async throws -> StockQuote
func fetchPortfolio() async throws -> [PortfolioItem]
func fetchNews(symbol: String) async throws -> [NewsItem]
func fetchDailyVolumes(symbol: String, days: Int) async throws -> [Int]
```

### Phase 4.4 — 키움증권 REST API 실구현

#### KiwoomAdapter

키움 Open API+의 REST 엔드포인트를 사용한다. Base URL: `https://api.kiwoom.com`

**토큰 발급**: `POST /oauth2/token` — `grant_type: client_credentials` 방식. 응답 `token_type`이 `"bearer"`(소문자)임에 주의.

**현재가 조회**: `POST /api/dostk/quot`
- `tr_cd: "STCA"`, `stk_cd: 종목코드`
- 응답 필드: `stck_prpr`(현재가), `prdy_vrss`(대비), `prdy_ctrt`(등락률), `acml_vol`(거래량)

초기 파싱 오류:
- `stck_prpr` 필드가 응답에 없어 빈 문자열 → `Int` 변환 실패
- 원인: 응답 JSON의 실제 필드 구조가 `output → stck_prpr` (중첩)이 아닌 플랫 구조
- 해결: `output` 키 없이 직접 최상위 필드로 파싱하도록 수정

**잔고조회**: `POST /api/dostk/acnt`
- `tr_cd: "kt00018"`, 계좌번호 파싱: `"-"` 제거 후 8+2 분리 (KIS와 동일 로직)

#### 관심종목 브로커 무관 정책

관심종목(`watchlist`)은 어느 브로커로 연결해도 동일하게 보이는 것이 자연스럽다는 결론으로, `accountId` 필터 없이 전체를 반환하도록 `fetchWatchlist()`를 변경했다. 포트폴리오는 계정 종속을 유지.

### Phase 4.5 — 계정 종속 관심종목·포트폴리오

**`AccountManager`**: Keychain 기반 계정 ID 관리 유틸리티.

```swift
enum AccountManager {
    static var connectedAccountIds: [String]   // ["KIS-ABCD1234", "KIWOOM-EFGH5678"]
    static var isAnyConnected: Bool
    static var currentAccountId: String        // 첫 번째 연결 계좌, 없으면 ""
}
```

계정 ID 형식: `"KIS-" + appKey.prefix(8)` / `"KIWOOM-" + appKey.prefix(8)`

**DB Migration v8**: `watchlist.accountId`, `portfolio.accountId` 컬럼 추가 (`DEFAULT ''`).

**기존 행 마이그레이션**: `AppDelegate.setupAdapter()` 및 `SettingsView.login()` 에서 `assignAccountIdToOrphanedItems(accountId:)` 호출. `accountId == ""` 인 행을 현재 로그인 계좌 ID로 일괄 업데이트. `UserDefaults "DB.v8AccountIdMigrated"` 플래그로 중복 방지.

### Phase 4.7 — 멀티 브로커 동시 모니터링

#### QuoteManager 구조 변경

단일 어댑터(`adapter`)에서 딕셔너리(`adapters: [String: any BrokerAdapter]`)로 전환.

```swift
func addAdapter(id: String, adapter: any BrokerAdapter)
func removeAdapter(id: String)
func fetchAll() async  // 등록된 모든 어댑터를 병렬로 fetchQuote
```

`fetchAll()` 내부에서 `await withTaskGroup(of: Void.self)`로 각 어댑터를 병렬 실행. 어떤 어댑터가 실패해도 나머지는 계속 진행 (실패는 로그로 기록).

**폴백 키 `"__primary"`**: 어댑터가 없을 때 `MockBrokerAdapter`를 `addAdapter(id: "__primary", ...)`로 등록.

#### AppDelegate 초기화 로직

```
KIS 자격증명 있으면 → KISAdapter addAdapter("KIS-XXXX")
키움 자격증명 있으면 → KiwoomAdapter addAdapter("KIWOOM-XXXX")
둘 다 없으면 → MockBrokerAdapter addAdapter("__primary")
```

#### DB 포트폴리오 조회 — 멀티 계정

`fetchPortfolio()`: `AccountManager.connectedAccountIds`가 비어있으면 빈 배열 반환. 있으면 `WHERE accountId IN (...)` 쿼리로 연결된 모든 계좌의 항목을 합산 반환.

#### 설정창 포트폴리오 탭 — 브로커 필터

`isMultiBroker`(연결 계좌 2개 이상)일 때 브로커 필터 메뉴 표시.

- 전체 선택: `Section` 헤더(`KIS` / `키움`)로 브로커 구분 표시
- 단일 브로커 필터링: 해당 브로커 항목만 flat 리스트로 표시

`isShowingAllBrokers` 조건:
```swift
isMultiBroker && (selectedBrokerIds.isEmpty || selectedBrokerIds.count == connectedBrokerIds.count)
```

### Phase 4.7.8 — 팝오버 UX 개선

#### 팝오버 포트폴리오 브로커 구분

`groupedHoldings` computed property: `AccountManager.connectedAccountIds` 순서대로 각 브로커의 `showInPopover` 항목을 그룹핑.

`brokerDivider(_:)`: `KIS ────` / `키움 ────` 스타일의 브로커 소헤더. `.tertiary` 색상 텍스트 + `0.5pt` 구분선 조합.

멀티 브로커 연결 상태에서 showInPopover 항목이 한 브로커에만 있더라도 소헤더를 표시해 어느 증권사 종목인지 알 수 있도록 함.

#### 포트폴리오 행 정보 표시

| 영역 | 표시 내용 |
|------|-----------|
| 좌측 위 | 종목명 (secondary) |
| 좌측 아래 (옵션) | 매입단가 · 현재단가 · 수량 (tertiary, 설정 토글 시 표시) |
| 우측 위 | 현재 평가금액 (`price × quantity`) |
| 우측 아래 | 평가손익 (`±XXX원`, green/red) |

#### 설정 연동 팝오버 토글

`@AppStorage` 키로 영속 저장, 설정 변경이 팝오버에 즉시 반영됨.

| AppStorage 키 | 설정 위치 | 팝오버 효과 |
|--------------|-----------|-------------|
| `Popover.showWatchlistDetail` | 관심종목 탭 → "팝오버 표시" | 종목코드·그룹 보조 텍스트 표시/숨김 |
| `Popover.showPortfolioDetail` | 포트폴리오 탭 → "팝오버 표시" | 매입단가·현재단가·수량 보조 텍스트 표시/숨김 |

#### 팝오버 외부 클릭 닫힘

`NSEvent.addGlobalMonitorForEvents(matching: [.leftMouseDown, .rightMouseDown])`로 글로벌 이벤트 모니터 등록. 팝오버 영역 밖 클릭 시 `closePopover()` 호출.

이벤트 모니터 생명주기:
- 등록: 팝오버 열릴 때 (`togglePopover`, `handleOpenPopover`)
- 해제: 팝오버 닫힐 때 (`popoverDidClose` 델리게이트 콜백)
- 중복 방지: `guard eventMonitor == nil`으로 이중 등록 차단

#### 빌드 후 바탕화면 아이콘 자동 생성

`project.yml`에 `postBuildScripts` 추가.

```bash
ditto "$BUILT_PRODUCTS_DIR/StockWatch.app" "$HOME/Applications/StockWatch.app"
# 최초 1회: 바탕화면에 심볼릭 링크 생성
ln -s "$HOME/Applications/StockWatch.app" "$HOME/Desktop/StockWatch.app"
```

빌드마다 `~/Applications/StockWatch.app`이 최신본으로 갱신되고, 바탕화면 링크는 항상 거기를 가리킨다.

## 버그 수정

### BackupManager 테스트 3건 실패 → 수정

**원인**: `AccountManager.connectedAccountIds`가 Keychain을 직접 읽는 구조여서, Keychain이 비어있는 테스트 환경에서는 빈 배열을 반환했다. `fetchPortfolio()`의 `WHERE accountId IN (...)` 쿼리가 빈 IN절이 되어 항목을 읽지 못함.

**수정**: `AccountManager`에 `#if DEBUG testAccountId` 오버라이드 추가.
```swift
#if DEBUG
nonisolated(unsafe) static var testAccountId: String? = nil
#endif

static var connectedAccountIds: [String] {
    #if DEBUG
    if let override = testAccountId { return [override] }
    #endif
    // ... Keychain 조회 ...
}
```

## 기술적 결정 사항

| 결정 | 이유 |
|------|------|
| 어댑터를 딕셔너리(`[String: any BrokerAdapter]`)로 관리 | 브로커별 독립 키 유지 → 한 브로커 실패가 다른 브로커에 영향을 주지 않음 |
| 관심종목은 브로커 무관 | 관심종목은 시세 조회용이라 어떤 브로커를 쓰든 동일한 목록이 자연스러움 |
| `.help()` 대신 `@AppStorage` 토글 | `NSPopover` 내에서 `_NSPopoverWindow` 구조 때문에 시스템 툴팁이 신뢰성 있게 표시되지 않음 |
| `postBuildScript`로 `~/Applications` 복사 | DerivedData 경로가 빌드마다 달라질 수 있으므로 고정 경로에 항상 최신본 유지 |
