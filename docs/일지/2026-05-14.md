## 작업 내역

### Phase 0 — 개발 환경 세팅

**목표:** 빌드 가능한 Xcode 프로젝트 구성

- Xcode 26.5 / Swift 6.3.2 / Homebrew 환경 확인
- **xcodegen** 도입 — `project.yml` 하나로 `.xcodeproj`를 재생성하는 방식 채택
  - 이유: `.xcodeproj`의 `project.pbxproj`는 팀 협업 시 충돌이 잦고, 파일 추가·삭제마다 수동 편집이 필요함. xcodegen을 쓰면 소스 디렉토리 전체를 자동 포함
- **GRDB.swift** 6.x를 SPM으로 추가 — SQLite ORM
- `Info.plist` 설정: `LSUIElement = YES` (Dock 아이콘 숨김, 메뉴바 전용 앱)
- Entitlements: App Sandbox 비활성화, Network Client 허용
- 코드 서명: Manual / ad-hoc (로컬 개발 전용, Apple 공증 없음)
- 초기 빌드 성공 확인 후 git 커밋

### Phase 1 — MVP

#### 1-1. 앱 기반 구조 (AppKit + SwiftUI 혼합)

macOS 메뉴바 앱은 SwiftUI만으로 구성하기 어렵다. `NSStatusBar`, `NSPopover`, `NSWindow`는 AppKit API이므로 **AppDelegate + SwiftUI View 혼합** 방식을 채택했다.

- `AppDelegate`: 메뉴바 아이콘 등록, 팝오버·설정 윈도우 생성 및 관리
- `MenuBarPopoverView` (SwiftUI): 아이콘 클릭 시 표시되는 300×320 팝오버
- `SettingsView` (SwiftUI): 탭 기반 설정 창 (580×460)

컴포넌트 간 통신은 `NSNotificationCenter`를 활용:
```
.openSettings    — 설정 창 열기
.openPopover     — 팝오버 열기 (알림 클릭 시)
.popoverWillShow — 팝오버 노출 시 데이터 새로고침
```

#### 1-2. 데이터 모델 및 SQLite 설계

GRDB의 `FetchableRecord` + `PersistableRecord` 프로토콜을 채택해 모델 → DB 매핑을 자동화했다.

| 테이블 | 역할 |
|--------|------|
| `watchlist` | 관심 종목 (코드, 이름, 별칭, 그룹) |
| `portfolio` | 보유 종목 (코드, 평균매입가, 수량) |
| `alert_conditions` | 알림 조건 (트리거 유형, 임계값, 쿨다운) |
| `alert_history` | 발생한 알림 이력 |

스키마 변경은 `DatabaseMigrator`로 버전 관리 — 한 번 등록된 Migration은 수정하지 않는 원칙을 적용.

DB 파일 경로: `~/Library/Application Support/StockWatch/stockwatch.db`

#### 1-3. 브로커 어댑터 아키텍처

증권사 API 교체 가능성을 고려해 **Protocol 기반 어댑터 패턴** 설계:

```swift
protocol BrokerAdapter: Sendable {
    var brokerName: String { get }
    func connect(credentials: BrokerCredentials) async throws
    func fetchQuote(symbol: String) async throws -> StockQuote
    func fetchPortfolio() async throws -> [PortfolioItem]
}
```

Swift 6 strict concurrency 환경에서 mutable state(토큰 캐시 등)를 안전하게 관리하기 위해 **어댑터를 `actor`로 구현**하는 규칙을 정했다.

현재 구현체:
- `MockBrokerAdapter` — 오프라인 개발용, 기준가 ±500원 랜덤 변동
- `KISAdapter` (actor) — 한국투자증권 REST API 실연동

#### 1-4. 한국투자증권(KIS) API 연동

**브로커 선택 배경:** PRD 초안은 키움증권 API를 기준으로 작성됐으나, 키움 영웅문 COM API가 Windows 전용임을 확인. KIS Developers API로 전환.

KIS REST API 연동 구현 사항:

| 항목 | 내용 |
|------|------|
| 인증 | `POST /oauth2/tokenP` — 액세스 토큰 발급, 24시간 유효 |
| 토큰 캐싱 | 만료 5분 전 자동 갱신, 401 응답 시 즉시 재발급 후 1회 재시도 |
| 시세 조회 | `GET /uapi/domestic-stock/v1/quotations/inquire-price` |
| 환경 전환 | 실전(`openapi.koreainvestment.com:9443`) / 모의(`openapivts.koreainvestment.com:29443`) 런타임 전환 지원 |

**트러블슈팅 기록:**

1. **403 Forbidden**: 토큰 발급 단계에서 403 발생. 원인 불명확한 채로 재시도 중 자연 해결됨. (KIS API 측 일시적 오류로 추정)
2. **DecodingError (keyNotFound)**: KIS API는 장 마감 시간 외에 일부 필드를 생략함. `KISQuoteOutput`의 모든 필드를 `String?`(옵셔널)로 변경해 해결.
3. **`prdy_vrss` 부호 처리**: 전일 대비 변동폭이 항상 양수로 오고 부호가 `prdy_vrss_sign` 별도 필드에 있음 (2=상승, 3=보합, 4=하락). 명시적 파싱 추가.
4. **`custtype` 헤더**: 초기에 `"J"` (법인)으로 설정했으나 개인투자자는 `"P"`여야 함. 수정.

#### 1-5. 자격증명 보안 (Keychain)

API 키를 `UserDefaults`나 파일에 평문 저장하는 대신 **macOS Keychain** 활용:

```swift
KeychainHelper.save(appKey, account: "kis.appKey")
KeychainHelper.load(account: "kis.appKey")
```

앱 시작 시 Keychain에 자격증명이 있으면 KISAdapter로 자동 연결, 없으면 MockBrokerAdapter로 폴백.

#### 1-6. 시세 폴링 및 알림 평가

```
QuoteManager (3초 폴링)
  └─ fetchAll() 완료 시
       └─ AlertEvaluator.evaluate()
            └─ 조건 충족 시 NotificationManager.send()
                 ├─ UNUserNotificationCenter (알림 센터 등록)
                 ├─ ToastWindowManager (화면 오버레이 배너)
                 └─ NSSound (시스템 사운드 재생)
```

폴링 결과로 `QuoteManager.connectionState` 업데이트:
- 성공 → `.connected`
- 2회 연속 실패 → `.error` → 메뉴바 아이콘 `⚠` 변경

#### 1-7. 설정 UI — 계좌 연결 탭 (로그인 개념)

일반적인 "저장" 버튼 대신 **로그인/로그아웃 개념**으로 설계:

**로그인 전 (폼):** AppKey / AppSecret / 계좌번호 입력 + 연결 테스트 버튼  
**로그인 후 (카드):** 연결 상태, 계좌번호, 마스킹된 앱 키, 로그인 시각 표시

앱 키 마스킹: `앞 4자리 + ●●●●●●●●●●●● + 뒤 4자리`

#### 1-8. 알림 클릭 → 팝오버 오픈 (1.9)

`UNUserNotificationCenterDelegate.didReceive` 구현:
```
알림 클릭
  → .openPopover 포스트
  → AppDelegate.handleOpenPopover()
  → NSApp.activate() + popover.show()
```
